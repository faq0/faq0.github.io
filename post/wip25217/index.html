新增内容

开发人员可以开始构建第三方小组件

最新版本支持第三方小组件，它是 WinAppSDK 1.2 Preview 2 版本更新的一部分。Windows 团队一直在努力将Windows 11中首次引入的小组件板功能扩展到更多开发人员。此版本中包含的第三方小组件平台将允许开发人员为 Win32 应用创建小组件，并在 Windows 11 小组件板上对其进行本地测试。

新功能：

  • 如果计算机已经打开了“开发人员模式”——即 Windows设置-隐私和安全性-开发者选项 中的“开发人员模式”选项、将 Windows 升级到了最新预览版、将小组件版本升级至 V521.20060.1205.0 或以上后，开发人员可以使用已打包的 Win32 应用在本地创建和测试第三方小组件。有关小组件的更多信息，包括其先决条件，请参阅小组件设计文档与小组件开发文档。

已知限制：

  • 第三方小组件只能在此预览版上进行本地测试。WinAppSDK 1.2 GA 发布后，Windows 11 正式版用户可以从微软商店获取该应用的第三方小部件。
  • 对第三方PWA小组件的支持也将作为未来 Microsoft Edge 版本更新的一部分。

简体中文 IME 中的云建议

更改和改进

[输入]

  • Build 25179 中新增了泰米尔语 Anjal 键盘，它现在可供所有 Dev 通道中的 Windows Insiders 使用。若要添加它，请确保泰米尔语（新加坡）、泰米尔语（马来西亚）、泰米尔语（斯里兰卡）或泰米尔语（印度）于 Windows设置-时间和语言-语言和区域 中列出 ，并单击“...”，选择“语言选项”，然后将“Tamil Anjal (QWERTY)” 添加到键盘列表中。
  • 我们已禁用于 Build 25188 中推出的新触摸键盘设置。在进一步完善设计后，将来可能将此功能重新加入。如之前的博客，在 Dev 或 Beta 频道中内测的功能并不一定会推送至正式版。

修复

[针对平板电脑优化的任务栏]

注意：仅当您的设备启用了平板电脑优化任务栏选项（已于此处记录）时，才会获得这些修复。平板电脑优化的任务栏仍在向 Windows Insiders 推出，还未全量化。

  • 修复了导致任务栏在没有正在运行的窗口时可能折叠的问题。
  • 修复了使用左边缘或右边缘手势时分别导致小组件、通知中心与任务栏重叠或看起来被任务栏截断的问题。
  • 修复了导致任务栏在切换应用时 explorer.exe 崩溃的问题。
  • 修复了打开浮出控件后 explorer.exe 崩溃的问题。

[窗口]

  • 修复了“任务视图”中窗口预览缩略图的关闭按钮略微超出缩略图范围的问题。
  • 修复了在最近版本的窗口中切换窗口，渲染时看到窗口闪烁的问题。
  • 修复了在互联网连接稳定时，导致 Miracast 视频质量在最近的版本中缓慢、断断续续的问题。

[任务栏更新]

注意：仅当您的设备上启用了任务栏更新（此处记录）时，才会获得这些修复。平板电脑优化的任务栏仍在向 Windows Insiders 推出，尚未全量化。

  • “快速设置”项目现在可以再次通过触摸进行重新排列。

[设置]

  • 修复了 Windows 设置-网络和 Internet -高级网络设置 下在某些网络中显示错误的连接速度的问题。

[其他]

  • 修复了 ARM64 电脑上的远程桌面意外地仅使用 TCP，不使用 TCP 的问题。
    注意：此处提到的某些修复可能会进入 Windows 11 正式版服务更新中。

已知问题

[总览]

  • 音频在升级最近版本后停止工作的问题正在被调查。
  • 应用在最近版本中崩溃的问题正在被调查。
  • 应用中的各种 UI 元素似乎在升级最新版本后消失，又重新出现的问题正在被调查。

[平板电脑优化的任务栏]

  • 在桌面姿态和平板电脑姿态之间切换时，任务栏有时会闪烁。
  • 在桌面姿态和平板电脑姿态之间切换时，任务栏需要更长的时间过渡到触摸优化版本。
  • 使用底部右侧边缘手势打开“快速设置”时，任务栏有时会停留在展开状态，而不是关闭到折叠状态。

[小组件]

  • 在从右向左显示语言（如阿拉伯语）中，当单击小组件板的展开视图时，内容会在小部件板调整大小之前以动画形式离开视图。
  • 通知标记编号可能未在任务栏上对齐。

新功能：Windows 11 中 Microsoft Teams Chat 功能的新视频通话体验

Windows 11 中的 Microsoft Teams Chat 功能将采用更直观，更简化的方式与世界上的任何人进行视频通话。

来自 Windows 11 中的Microsoft Teams Chat 功能的新视频通话体验


当您从任务栏打开 Chat 功能时，窗口焦点将放在您身上，您可以预览自己的摄像头展示效果，以及与人员联系的各种方式。现在，与其他使用 Teams 个人版的人进行视频通话更加方便。对于那些不在 Teams 中的人来说，这也不是问题——您可以轻松地通过短信或电子邮件与任何人共享呼叫链接，并直接使用 Chat 功能进行对话。 这种体验将在未来几个月内推广到更多成员，但一小部分用户现在已经可以开始体验。从任务栏启动 Chat 功能来查看您是否被加入。您可以直接在该功能中提供反馈，我们很乐意倾听您的反馈。 *反馈：请在反馈中心（WIN + F）中的“桌面环境”>“聊天”下提交反馈。* ## Microsoft Store 更新 微软团队开始推出微软应用商店更新（版本 22209），其中包含以下改进： + 在应用商店中查找游戏时，现在清楚地显示它是否是 Xbox Game Pass 订阅中的游戏，以及当前的价格。在此类游戏的详细信息页面上，我们更新了按钮设计并添加了一个新部分，让您知道订阅中包含的内容。
应用商店中的游戏显示它是否包含于 Xbox Game Pass 订阅，以及当前的价格


+ 我们通过简化的选项和改进的性能增强了“库”选项卡的体验。 *反馈：请在反馈中心 （WIN + F）中的“Microsoft Store”下提交反馈。*

关于开发频道
开发人员频道接收的构建代表了我们工程师的长期领导工作，这些工程师的功能和经验在我们尝试不同的概念并获得反馈时可能永远不会发布。请务必记住，我们发布到开发人员频道的内部版本不应被视为与任何特定版本的 Windows 匹配，并且所包含的功能可能会随着时间的推移而更改、删除或在预览体验成员版本中替换，或者可能永远不会在 Windows 预览体验成员之外向一般客户发布。有关详细信息，请阅读此博客文章，了解我们计划如何使用开发频道来孵化新想法、处理长线索项目以及控制各个功能的状态。

这些并不总是稳定的版本，有时您会看到阻止关键活动或需要解决方法的问题。请务必阅读我们博客文章中列出的已知问题，因为我们记录了每个航班中的许多此类问题。

开发人员频道中的内部版本号高于测试版和发布预览版频道中的 Windows 11 预览版。如果不执行全新安装，则无法从开发人员频道切换到测试版或发布预览版，否则无法将其安装回当前发布的 Windows 11 版本。

另外：由于开发和 Beta 渠道代表了我们工程师的并行开发路径，因此在某些情况下，功能和体验可能会首先显示在测试版渠道中。

您在桌面右下角看到的桌面水印对于这些预发行版本是正常的。

关于Dev 渠道

Dev 通道收到的更新代表工程师的长期工作，其中包含了可能永远也不会推送至正式版更新的功能。Dev 通道也是我们尝试不同的概念并获取反馈的版本。请务必记住，推送至 Dev 通道的更新不应该与任何 Windows 版本相匹配。其中的功能可能会随着时间的推移而更改、移除、在 Insider 版本中被替代、或可能永远不会向一般客户发布。有关详细信息，请阅读此博客文章，了解我们计划如何用 Dev 通道孵化新想法、处理长期项目以及控制各个独立功能的状态。

这些版本并不总是稳定版本，有时您会看到阻止关键活动或需要解决方法的问题。请务必阅读 Windows Insider Blog 文章列出的已知问题，因为它记录了每个版本中的许多此类问题。
目前，Dev 通道的版本号高于 Beta 与 Release Preview 通道。您将需要对设备执行全新安装回正式版后才能切换至 Beta 与 Release Preview 通道。

另外：由于 Dev 和 Beta 渠道代表了工程师的并行开发路径，因此在某些情况下，功能和体验可能会首先显示在 Beta 通道中。
