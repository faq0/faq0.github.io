新增功能

更新管理改进

确保 Windows 更新的重启时间一致且可预测是成功更新策略的关键部分。为了提高重启的可预测性，我们对更新堆栈业务流程协调程序 (Update Stack Orchestrator) 进行了以下更改：

  • 质量和功能更新的截止时间计算现在基于客户端的更新扫描最初发现更新的时间。
    以前，截止时间基于质量更新的更新发布日期和功能更新的重启时间。
  • "指定自动更新和重启的截止时间" 组策略设置拆分为两个单独的设置，以与现有配置服务提供程序 （CSP） 策略保持一致。组策略设置中的选项对于功能和质量更新是相同的。每个新设置还具有 "宽限期结束前不自动重启" 选项。新的组策略设置包括：
    • 指定自动更新的截止时间与质量更新的重新启动截止时间
    • ** 指定自动更新的截止时间与需重新启动的功能更新的截止时间**
  • 为了减少截止时间计算更改的影响，功能更新的默认值已更改为以下内容：
    • Update/ConfigureDeadlineForFeatureUpdates = 2（以前为 7）
    • Update/ConfigureDeadlineGracePeriodForFeatureUpdate = 7（以前为 2）

反馈：请在反馈中心（WIN + F）中“安装和更新”下的“所有其他问题”下提交反馈。

更改和改进

[开始菜单]

  • 我们正在尝试对开始菜单进行一个小的更改，一些 Windows Insiders 将在其用户配置文件上看到通知标记，通知他们需要执行某些操作。

[小组件]

  • 我们正在尝试标题栏的一些不同位置，其中包含小组件板的新图标，以了解如何为我们的用户最好地优化小组件板。通过使用单行图标，用户可以得到一致、干净且易于访问的图标，并通过简单的视觉效果传达动作和功能。

[输入]

  • 从 Build 25206 中开始推出的密码字段上使用剪贴板历史记录 （WIN + V） 的更改现在可供 Dev 通道中的所有 Windows Insiders 使用。
  • 你可以再次向 Microsoft 提供语音剪辑，即使使用了设备上的语音识别也是如此。

[语音输入]

  • 在使用同一 Microsoft 帐户登录时，所有设备上都能同步语音键入设置、自动标点符号和语音输入启动器。您可以通过 设置>帐户>Windows 备份>记住我的偏好>辅助功能 来切换此功能。这目前仅适用于当前的 Microsoft 帐户，以后将支持 AAD 账户。

[设置]

  • 如果用户选择在 系统>存储>磁盘和卷>磁盘属性 下更改存储驱动器的分区样式，则新对话框将警告用户潜在的数据丢失。
  • 我们更新了设置应用中的以太网和 Wi-Fi 属性页，现在在网络详细信息摘要中可以显示网关信息。

修复

[总览]

  • 修复了导致某些 Windows Insiders 在升级到最新的 Dev 通道版本时出现 0xC1900101 错误。
    *这是一个通用的回滚错误代码，因此如果您在更新后继续看到此错误，您可能会遇到其他问题。如果发生这种情况，请在反馈中心的“安装和更新”>“下载、安装和配置 Windows 更新”下提交有关它的反馈。

[平板电脑优化任务栏]

注意：仅当设备上启用了平板电脑优化任务栏（此处记录）时，才会显示这些修复。平板电脑优化的任务栏仍在向 Windows Insiders 推出，尚未全量化。

  • 修复了在平板电脑优化任务栏上的应用之间切换时导致 explorer.exe 崩溃的问题。
  • 开始屏幕的触摸手势在使用平板电脑优化的任务栏时应正确地跟随手指。
  • 改进了使用平板电脑优化任务栏的触摸手势及其动画与触摸键盘配合使用的方式。
  • 现在，您可以使用笔从任务栏中向上轻扫以展开平板电脑优化的任务栏。
  • [任务栏更新]
    注意：仅当设备上启用了任务栏更新（此处记录）时，才会显示这些修复。平板电脑优化的任务栏仍在向 Windows Insiders 推出，尚未全量化。
  • 修复了在任务栏中拖动图标时导致 explorer.exe 崩溃的问题。
  • 右键单击任务栏图标不应再导致自动隐藏的任务栏不正确地隐藏。
  • 任务栏中的显示隐藏图标面板不应阻止之后打开的上下文菜单。
  • 修复了首次尝试从任务栏打开“快速设置”时无法正常工作的问题。
  • 修复了阻止任务栏图标实时更新的问题。

[文件资源管理器]

  • 当您使用鼠标滚轮滚动到导航窗格的底部并尝试向上滚动时，它应该不再卡在底部。
  • 修复了在打开“打开文件”对话框或“保存文件”对话框时可能导致某些应用挂起的问题。
  • 对可能导致 Windows Insiders 在导航窗格中看到分隔线重叠条的情况进行了另一个修复。

[开始菜单]

  • 修复了在打开的开始菜单文件夹中拖动内容可能会导致崩溃的问题。

[搜索]

  • 进行了修复，以帮助解决单击任务栏中的搜索图标时搜索窗口可能没有响应的问题。
  • 从搜索中打开图像不应再意外地在“打印”中打开它们。

[输入]

  • 修复了您可能在一段时间后无法使用鼠标与某些游戏进行交互的问题。
  • 修复了乘法符号 （×， U+00D7） 在具有高分辨率屏幕的用户的 Yu Gothic 字体系列中无法正确显示的问题。

[设置]

  • 应用>启动应用 下的 UWP 应用图标的大小现在应显示正常的大小。

[任务管理器]

  • 从标题栏底部区域拖动任务管理器以移动窗口的操作现在应可以正常使用。

[其他]

  • 修复了导致某些应用在最新版本（包括 OneNote）中崩溃的几个问题。我们将继续调查反馈。
  • 进行了更改以解决在使用利用 Microsoft-Windows-DxgKrnl ETW 提供程序的性能监视工具时可能导致性能下降的问题。这也影响了Windows混合现实的性能。* 如果您遇到性能问题，请参阅此处，了解有关提交可操作的效果反馈的步骤。
  • 修复了启用了“收听此设备”的录音设备在播放声音时可能不会显示在音量混合器中的问题。
  • 如果重置电脑，则重置时显示的屏幕现在将为黑色，与升级屏幕一致。
  • 如果在以前的 Dev 通道版本中尝试玩某些游戏时遇到游戏服务崩溃，请查看 Microsoft Store 的游戏服务更新，现在应该已解决。
  • 修复了导致某些 PC 在升级到最新版本时意外进行错误检查的问题，错误消息为“MULTIPROCESSOR_CONFIGURATION_NOT_SUPPORTED”。
  • 修复了上一个版本中可能导致 Miracast 无法连接的问题。
    注意：此处在 Dev 通道的版本中记录的某些修复可能会进入 Windows 11 正式版的服务更新。

关于Dev 渠道

Dev 通道收到的更新代表工程师的长期工作，其中包含了可能永远也不会推送至正式版更新的功能。Dev 通道也是我们尝试不同的概念并获取反馈的版本。请务必记住，推送至 Dev 通道的更新不应该与任何 Windows 版本相匹配。其中的功能可能会随着时间的推移而更改、移除、在 Insider 版本中被替代、或可能永远不会向一般客户发布。有关详细信息，请阅读此博客文章，了解我们计划如何用 Dev 通道孵化新想法、处理长期项目以及控制各个独立功能的状态。

这些版本并不总是稳定版本，有时您会看到阻止关键活动或需要解决方法的问题。请务必阅读 Windows Insider Blog 文章列出的已知问题，因为它记录了每个版本中的许多此类问题。
目前，Dev 通道的版本号高于 Beta 与 Release Preview 通道。您将需要对设备执行全新安装回正式版后才能切换至 Beta 与 Release Preview 通道。

另外：由于 Dev 和 Beta 渠道代表了工程师的并行开发路径，因此在某些情况下，功能和体验可能会首先显示在 Beta 通道中。
