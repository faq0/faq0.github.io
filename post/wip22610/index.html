基本信息

22H2(NT10.0)

  1. 新的组策略，可以在Windows 11 SE中隐藏开始菜单中的“建议”部分

TL; DR

  • 此版本中包含新的对于IT管理员的组策略和对于家庭安全小组件的更新。
  • 此版本中还含有可以改善您的整体体验的修复。
  • 此版本中包含两个会影响Windows Insiders的已知问题。新的已知问题将被标为“[NEW]”。其中的一个已知问题将影响任务栏中的“显示隐藏应用”浮出控件。
  • [更新] 此版本中将不会发布对应的SDK。

Build 22610 中更新了什么？

面向 IT 管理员的其他新 MDM 和组策略

我们正在引入新策略，以便 IT 管理员可以在开始菜单、任务栏和系统托盘中简化Windows 11体验。
以下是推出的组策略：

  • 禁用快速设置浮出控件
  • 禁用通知中心和日历浮出控件
  • 禁用所有任务栏设置
  • 禁用搜索（在开始菜单和任务栏中）
  • 从任务栏中隐藏任务视图
  • 阻止在开始屏幕中自定义固定图标
  • 从“开始”菜单中删除“建议”部分
    通过此策略，您可以阻止"开始"菜单显示建议的应用程序和文件列表。
    如果启用此策略设置，"开始"菜单将不再显示包含推荐文件和应用列表的部分。
    适用于：Windows 11 SE
  • 禁用开始菜单中的上下文菜单
  • 在开始菜单中隐藏“所有应用”
    若要在本地配置这些新的组策略，请打开组策略编辑器，然后导航到“用户配置”>“管理模板”>”’开始’菜单和任务栏”。
    您也可以通过Microsoft Endpoint Manager部署这些策略。[https://docs.microsoft.com/mem/intune/configuration/custom-settings-configure]
    反馈：若要反馈，请在反馈中心(WIN + F) 中定位到：
  • 桌面环境>“开始”菜单，用于显示与“开始”相关的策略。
  • 桌面环境>任务栏，用于与任务栏相关的策略。
  • 桌面环境>与快速设置相关的策略的快速设置。
  • 桌面环境>通知中心，用于查找与通知中心相关的策略。

“家庭安全”小组件更新

为了帮助你和你的亲人保持安全和受到保护，我们将推出对“家庭安全”小组件的更新，通过新的位置共享视图为所有家庭成员提供改进，该视图显示你的家庭成员使用“家庭安全”应用的位置。此外，还有一项针对成员角色人员的新更新，其中包括跨应用和设备的屏幕时间使用情况的改进视图。
若要了解有关 Microsoft 家庭安全设置的详细信息并获取移动应用，请访问[https://aka.ms/microsoftfamilysafety]。请务必也查看Microsoft Family应用程序。

反馈：请将反馈提交在反馈中心 (WIN + F) 的 桌面环境 > 小组件 部分。

更改和改进

[总览]

  • 默认情况下，Windows 11 家庭版开发和测试版通道版本不再默认启用SMB1客户端。如果已手动安装SMB1或从安装了 SMB1 的旧版Windows升级，则升级后将不会删除 SMB1。
    有关更多信息，请在此处查看此博客文章。
    我们更新了锁定屏幕上电池图标的设计，使其与任务栏中的图标一致。
  • 如果你的电脑支持，我们现在将在具有最新Insider Preview版本的系统托盘中电池图标的提示文字中显示预估电池剩余时间。
  • 桌面右下角的版本水印在此版本中不再存在。这并不意味着我们已经完成了开发，水印将在未来的版本中返回。

[任务栏]

  • 根据 Windows 预览体验成员的反馈，我们禁用了平板电脑优化的任务栏功能，该功能目前已开始在 Build 22563 中向 Windows 预览体验成员推出。我们希望在进一步完善体验后，将来能将此功能带回来。正如我们之前提到的，我们在 Dev 或 Beta 频道中试用的功能可能并不会发布。
    [小组件]
  • 在Build 22563中发布的可以使得小组件带来更多实时内容的更新，将在开发和测试版渠道中向更多 Windows Insiders推出。

[任务管理器]

  • 我们更新了任务管理器中的“进程”页面，以便热图中使用的颜色现在基于您的主题色。请注意，有时我们将使用默认的蓝色主题色以确保可读性。我们在本月的 Windows Insider Preview网络直播中进行了演示，演讲人包括任务管理器团队的 Claton Hendricks – 在此处观看网络直播(Youtube链接)。
    任务管理器中的“进程”页面现在根据主题色显示颜色。

[文件资源管理器]

  • 我们更新了右键菜单和命令栏中使用的重命名、属性位置和图标，以提高可发现性和一致性。

[设置]

  • “你的手机”名称现在被更改为“手机连接”，位于设置>蓝牙和设备中。“你的手机”应用现在是“手机连接”官方博客 - 在此处阅读公告。

[其他]

  • 根据 Windows Insiders的反馈，我们将回滚在Build 22557 中的更改，其中不再允许 TKIP 和 WEP 连接。从此版本开始，将再次允许这些连接。当连接时给出警告，通知这不是受保护的连接。
    提醒一下，TKIP 和 WEP 是已弃用的安全标准，计划在将来的版本中删除。我们建议您将路由器或其他设备更新为至少支持 WPA2 的设备（WPA3 是当前标准）。

修复

[总览]

  • 我们修复了导致运行 Windows 10 的 Windows Insiders在尝试下载最新Dev/Beta版本时遇到下载错误代码0xc8000402的问题。如果您遇到此问题，请安装 KB5011831，然后再尝试下载最新版本。如果还有此问题，请在反馈中心提交反馈。
  • 我们修复了导致 Explorer.exe 在反复崩溃（并且无法成功加载）的问题，只有一小部分Windows Insiders受到影响。
  • 解决导致某些Windows Insiders在打开通知中心时遇到错误检查BAD_POOL_CALLER的问题。
  • 打开组策略编辑器时，将不再弹出管理模板错误。
  • 修复了在某些情况下，组策略编辑器在ARM64计算机上上意外显示“MMC无法初始化管理单元”的问题。

[任务栏]

  • 修复了阻止 Shift + 右键单击任务栏图标打开上下文菜单的问题。
  • 更改第二显示器的分辨率不应再导致任务栏应用图标意外向右移动。
  • 单击任务栏应立即关闭“快速设置”和“通知中心”等内容（如果它们已打开）。
  • 如果拔下第二监视器的插头，则第二监视器上使用的 Windows 和任务栏图标现在将正确传输到主监视器。
  • 修复了系统图标（如“开始”、“搜索”和“任务”视图）上无法出现任务栏图标动画的问题。
  • 现在，动画将在您第一次拖放到任务栏时正确播放。
  • 如果以从右向左（RTL）语言拖动到任务栏，则该图标现在将显示在正确的一侧。
  • 连接到 Wi-Fi 后，任务栏中的网络图标将不再错误地显示为已断开连接。
  • 修复了在转换为夏令时时使任务栏时钟时发生崩溃的问题。
  • 现在，当您再次单击远离“显示隐藏的图标”的浮出控件时，它将会关闭。
  • 当您将系统托盘中的所有图标设置为显示时，将不再显示空的“显示隐藏图标”箭头。
  • 修复了在小屏幕上打开“通知中心”时被切断的问题，即隐藏“请勿打扰”按钮。
  • 修复了阿拉伯语和希伯来语显示语言的问题，即通知中心在关闭时会意外弹出。
  • 如果您的麦克风正在被使用，但定位未被使用，则将鼠标悬停在任务栏中的麦克风按钮上不应再在提示框中显示具有空应用名称的位置部分。(即%1)

[开始菜单]

  • 像Microsoft Edge这样的应用程序图标将不再在“开始”屏幕的所有应用程序列表的底部被切断。
  • 如果在拖动图标以在“开始”屏幕中创建文件夹时取消，则它们将不再卡在只有一个图标的文件夹中。
  • “所有应用”下的“Windows 工具”条目现在应再次可见。
  • “开始”菜单中文件夹的动画将在文件夹所在的位置启动，而不是第一个固定应用上。

[搜索]

  • 我们做了一些工作来帮助解决有时无法搜索刚安装的应用的问题，即使它显示在“开始”屏幕中。

[文件资源管理器]

  • 在文件资源管理器或桌面上使用重命名、复制、粘贴、删除和其他选项的右键菜单选项不应再导致explorer.exe在右键菜单关闭时崩溃。
  • 右键单击文件现在将显示文件资源管理器中更新的命名，并显示“添加到收藏夹”。
  • 在“打开或另存为”对话框的导航窗格中右键“此电脑”，现在应该会再次显示右键菜单。
  • 解决了在最近版本中选择大量文件时，文件资源管理器冻结的问题。
  • 我们对OneDrive文件夹中图像和压缩文件的键盘选择性能进行了一些改进。
  • 修复了在安装 WSL 时导致文件资源管理器频繁冻结的问题，以及通过“主文件夹”打开WSL中的文件的问题。
  • 为了提高性能，仅在打开“主文件夹”时，图像和视频现在将显示缩略图。
  • 更新了文件资源管理器中“Please sign in”按钮的提示框，从“Sign in again to fix authentication issues.”更改为“One of your accounts needs attention to show files from Office.com in Home”，使用途更清晰。
  • 我们解决了文件资源管理器中的“登录”按钮可能不起作用的问题。
  • 修复了在“主页”中加载最近使用的Office文件导致的explorer.exe崩溃问题。
  • 解决了右键单击某些文件类型时“打开”选项意外显示通用图标的问题。
  • 修复了一个潜在问题，该问题被认为是导致某些Windows Insiders在文件夹选项中的“Open File Explorer to”下拉列表中看不到“此电脑”选项的原因。

[输入]

  • 修复了以下几个问题：在某些情况下，表情符号面板、剪贴板历史记录和语音键入将在解锁电脑后或 Windows更新后立即无法使用。
  • 修复了物理键盘的文本建议有时可能以透明背景显示的问题。
  • 为了确保文本输入始终可用，如果在OOBE、登录屏幕或UAC对话框中使用触摸并点击文本字段，则即使连接了物理键盘，触摸键盘现在也将始终显示。
  • 锁定电脑时，如果在“设置”>“时间和语言”>”输入”>”触摸键盘”下关闭了“在我输入内容时播放按键音”，则在登录屏幕上使用触摸键盘时，不应再听到按键的声音。
  • 当使用德语触摸键盘时，按住引号键后添加了”排版引号”选项。
  • 如果你在平板电脑上使用停靠的触摸键盘并点击了应用中没有焦点的文本字段，触摸键盘现在不会出现并立即消失
  • 改进了启动语音键入 （WIN + H） 后进入侦听状态的性能。
  • 修复了在手写面板中使用删除手势删除字母时出现的崩溃问题。
  • 解决了如果在“设置”>“蓝牙和其他设备”>”触摸板“>”触摸板“下取消选中”连接鼠标时让触控板保持打开状态“后在连接->断开连接鼠标后不重新启用触摸板的问题。
  • 修复了在某些情况下，在屏幕上滑动手指时导致轨迹变红或不可见的问题。
  • 如果未安装中文（简体）IME，则“设置”>个性化>“文本输入”>“主题”下将不会再显示自定义主题。
  • 如果连接了 Xbox 控制器，则使用它与游戏交互不应再弹出 “No supported apps in use for mic mute.”的窗口

[小组件]

  • 改进了在升级或应用程序更新后首次登录时打开小组件板的可靠性。
  • 我们解决小部件板无法轻松关闭的问题。
  • 为了使状态更清晰，如果任务栏上的天气状态不可用，我们现在将显示一个灰色的天气图标。
  • 解决了导致升级后任务栏和设置中缺少”小组件”的潜在问题。此修复应使将来的电脑受到影响 - 如果你现在受到影响，请尝试重新安装这个应用(Windows Web Experience Pack)，可能会解决。

[设置]

  • 修复了英语（英国）的 Windows Update显示文本，使其显示“last checked”而不是“last ticked”。
  • 解决了导致在扫描模式下使用“讲述人”时无法激活 Windows 更新中的“立即重新启动”按钮的问题。
  • 如果您打开“快速设置”的“蓝牙”部分并返回，现在将不会导致“快速设置”中的滚动条样式发生变化。
  • 应用不应再在“设置”>”个性化“>”任务栏”>”其他系统托盘图标”下显示不正确的图标。
  • 修复了导致“设置”>”个性化“>”任务栏”下的搜索图标颠倒显示，并且在”Desert“对比度主题中显示错误颜色的问题。
  • 修复了“快速设置”无法使用触摸输入的问题。
  • 解决了导致“快速设置”上方的媒体控件在媒体源关闭后仍然保留的问题。
  • 修复了在导航到 “快速设置”中选择声音输出选项时的崩溃问题

[窗口化]

  • 解决与屏幕顶部的贴靠布局相关的问题，其中一个大的不可见区域可能会卡住，并会消耗该区域的点击次数。
  • 当您在最大化窗口的标题栏上向下拖动时，现在从屏幕顶部调用捕捉布局应该更容易。
  • 如果贴靠组中有一个窗口设置为在所有桌面上显示，则当您将贴靠组拖放到任务视图中的其他桌面时，该设置现在应保持不变。
  • 修复了在使用分屏时导致explorer.exe崩溃的问题。
  • 修复了最近版本中DWM崩溃的问题。
  • 解决了使用 3 指触摸板手势调出 ALT + Tab 时未正确设置焦点的问题。
  • 修复了当您只打开了一个 Microsoft Edge 窗口并尝试捕捉它时，贴靠助手意外出现的问题。
  • 减少了在拖动窗口时看到亚克力背景明显滞后的可能性。
  • 改进了在全屏使用应用时从屏幕边缘使用手势（例如打开“开始”菜单）的可靠性。
  • 解决了在使用3指在屏幕上的手势以最小化某些全屏应用程序顶部打开的应用程序后，可能导致屏幕显示卡住的问题。

[讲述人]

  • 修复了使用“讲述人”和“键盘导航“的”显示隐藏的图标”面板时出现的问题，其中焦点跳出到图标末尾的任务栏，而不是循环回到开头。
  • “讲述人”现在将从 Microsoft Edge 上页面的开头开始阅读，即使页面上的控件默认具有焦点（如必应的搜索框）。

[任务管理器]

  • 修复了导致某些应用在任务管理器中显示为已挂起的根本问题，即使它们本应已挂起。
  • 修复了展开子进程的子进程会导致进程中“名称”列的内容明显偏移的问题。
  • 在设置中单击“显示所有进程的历史记录”不应再挂起或崩溃任务管理器。
  • “运行新任务”按钮现在应该在“详细信息”和“服务”页上工作。
  • 为了解决文本和 UI 剪辑的问题，任务管理器将不再在标题栏上显示主题色。

[网络]

  • 当某些VPN连接时，导致某些Windows Insiders出现互联网连接问题的问题现在应该得到解决。

[锁定屏幕和登录]

  • 如果在启用对比主题时单击电源按钮，登录屏幕应该不会再崩溃。
  • 修复了登录屏幕上的网络图标在使用“设置”>辅助功能“中的文本缩放选项时可能会被裁剪的问题。
  • 解决了快速点击登录屏幕崩溃的问题。

[其他]

  • 解决了在 Windows 11 的初始化设置（OOBE） 中阻止辅助功能浮出控件显示或用鼠标选择其中的项目的几个问题。
  • 在初始化设置（OOBE） 中的RTL（从右到左）语言中，辅助功能浮出控件将不再为空。
  • 修复了导致通知弹出窗口卡在屏幕侧面的透明框以阻止输入的问题。
  • 解决了一个潜在的问题，该问题被认为是系统在最近连接UPS时认为您使用的是电池的根本原因。
  • 用于启动快速助手（WIN + CTRL + Q）的键盘快捷方式现在应该再次工作。
  • 解决了某些已签名的应用程序被智能应用程序控制意外阻止的问题。
  • 如果使用键盘控制亮度或音量级别，则在“设置”中关闭动画效果时，现在显示的浮出控件为”设置”>辅助功能>”视觉效果”。
  • 解决了在使用 Xbox 游戏录制工具栏截取屏幕截图后可能导致黄色边框卡在游戏周围的潜在问题。
  • 修复了导致 Windows 安全中心应用在升级后未启动或显示提示，指出该协议未安装应用的基础应用注册问题。
  • 进行了更改，该更改应有助于解决预览体验成员看到 Windows 终端无法在 Microsoft Store 中更新并出现错误代码0x80073CFB的问题。
  • 修复了导致依赖 .NET Framework 的某些应用在最后几次运行中无法启动的问题。
  • 做了一些工作来解决 uxtheme.dll中的泄漏，导致某些应用中的性能随时间推移而有所下降。
  • 解决了导致碎片整理程序显示错误的问题，指出“目前不支持（0x80004002）此类接口”。
  • 修复了预览体验成员在断开蓝牙设备或其他蓝牙方案时可能遇到的最近版本中的错误检查错误DPC_WATCHDOG_VIOLATION。
  • 改进了连接 Apple AirPods 等蓝牙设备的可靠性。
  • 修复了连接某些蓝牙耳机后，从休眠状态恢复后导致音频无法立即播放的问题。
  • 解决了在最近版本中连接某些 USB 音频设备时可能导致错误检查并出现错误PAGE_FAULT_IN_NONPAGED_AREA的问题。
  • 修复了上一个版本中与 cdp.dll 相关的崩溃问题。
  • 修复了导致无线显示应用在过去几个版本中无法接收内容的潜在问题。

已知问题

[总览]

  • [NEW]升级到Build 22610后，系统托盘中的“显示隐藏的图标”浮出控件可能会完全消失，即使它在设置>个性化>任务栏>其他系统托盘图标下显示为“打开”。我们正在调查此问题。如果你需要访问将在“显示隐藏的的图标”浮出控件中显示的图标，你可以右键单击任务栏以打开任务栏设置页面，导航到“其他系统托盘图标”，然后把该应用切换为“开”，直到问题得到解决。
    非官方解决办法：重启explorer.exe

[任务管理器]

  • [NEW]从任务管理器的命令栏中启用效率模式可能会导致进程列表在“进程”页面上的“应用程序”和“后台”组之间波动。解决方法：使用右键菜单启用/禁用效率模式，然后重新打开任务管理器。

[小组件]

  • 从屏幕侧面使用手势时，小组件可能无法打开。请暂时点击任务栏中的“小组件”图标。

[实时字幕]

  • 某些全屏应用（例如视频播放器）会阻止实时字幕可见。
  • 某些位于屏幕顶部附近且在运行实时字幕之前关闭的应用将在位于顶部的实时字幕窗口后面重新启动。在应用具有焦点时使用系统菜单（ALT + 空格键）可将应用的窗口进一步下移。

关于Dev和 Beta渠道 [已更新!]

提醒一下，我们正在改进开发和发布给Windows Insiders的WIP Dev和Beta渠道的方式，Dev和 Beta 渠道现在代表了我们工程师的并行开发路径。Dev渠道将是一个我们将尝试不同概念，产生新想法以及处理可能无法发布给普通客户的长周期项目的地方。Beta 版渠道中，我们将会发布更加接近普通客户的体验的版本。但是，这并不意味着我们在 Beta 版频道中试用的每个功能都将正式发布。Windows Insiders可以阅读上个月的这篇博客文章，其中概述了我们将在Dev和 Beta 渠道中与Windows Insiders一起尝试新功能的方式。
现在，Dev 和 Beta 版频道正在接收相同的内部版本。因此，如果您希望从Dev渠道切换至Beta渠道，请按照以下简单步骤进行切换：

  1. 进入设置 > Windows 更新 > Windows 预览体验计划。
  2. 选择“选择预览体验成员设置”。
  3. 选择Beta渠道。
  4. 下次您收到更新时，您将收到Beta渠道的更新。
    重要说明：一旦我们向Dev渠道推送更高版本号的更新，此通道将关闭。如果你的设备保留在Dev渠道上，并且收到的内部版本号高于Beta渠道中的内部版本号，则必须在设备上进行全新安装才能切换到Beta渠道。
    这些并不总是稳定的版本，有时您会看到阻止关键活动或需要解决方法的问题。请务必确保您阅读了博客文章中列出的已知问题，因为我们记录了每个内测版本的许多此类问题。
    您在桌面右下角看到的桌面水印对于这些预发布版本是正常的。
